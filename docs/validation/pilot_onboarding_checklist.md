# Trusted User Onboarding Checklist (Pre-Alpha)

Goal: onboard **3-5 trusted users** before Tuesday pilot handout.

## Candidate criteria
- Personally trusted relationship
- Comfortable giving blunt feedback
- Will run at least 2 debriefs/day for one week
- Understands this is pre-alpha quality software

## Onboarding steps (per user)
1. Confirm consent for AI-assisted transcript + summary generation.
2. Create user record in `docs/validation/pilot_users.csv`.
3. Share access URL and quick-start workflow (record -> follow-up -> generate -> download).
4. Ask for first-run completion within 24 hours.
5. Collect first trust/cognitive-load score via feedback form.

## Success gate
- At least 3 users marked `active`
- At least one completed debrief per active user
- No critical cross-user data leak observed

## Timeline
- Monday: onboard + smoke
- Tuesday: handout readiness checkpoint
